// How to Walk Through This Guide

01 Start with TF Playground and Interactive ML — these are zero-install, run in your browser right now. Build a feel for how neural nets, loss surfaces, and classifiers behave before reading anything deeper.
02 Move to the Poloclub explainers — CNN Explainer, Transformer Explainer, and Diffusion Explainer each run a live model in your browser. They are the fastest way to truly internalize how these architectures work.
03 Read the Distill.pub articles for the concepts that still feel fuzzy. They are peer-reviewed, interactive, and written by the same researchers who built the models — nothing else comes close for depth.
04 Star the GitHub repos — ML-Tokyo/Interactive_Tools is the master list. Reference it whenever you are about to build something from scratch that someone has already visualized.

🧠 Foundations
5 Tools for Neural Network Intuition

These are your first stops. Run them before reading any theory. Watching a neural network learn in real time gives you intuition that no amount of textbook reading can replicate. All browser-based, zero install.

01 TensorFlow Playground → OPEN
Neural Nets Interactive
Draw your own dataset. Watch a network learn it in real time.
The canonical starting point for understanding neural networks. You pick the dataset (spirals, circles, XOR, Gaussian), define the architecture (layers, neurons, activation), set the learning rate and regularization — then hit play and watch the decision boundary form epoch by epoch.

How to use it: Try to classify the spiral dataset with one hidden layer and watch it fail. Then add layers and watch it succeed. That failure-then-success experience is the intuition for why depth matters.
// Best For
Building raw intuition for activation functions, overfitting, depth vs width, and why regularization exists — before reading a single equation.
02 Interactive ML Platform → OPEN
Neural Nets Math Interactive
30+ live demos covering the full ML curriculum — sliders everywhere.
The most comprehensive single site for hands-on ML learning. Covers linear regression, logistic regression, regularization, decision trees, SVMs, k-means, DBSCAN, gradient descent on 3D loss surfaces, bias–variance tradeoff, ROC curves, PCA, k-fold cross validation, confusion matrices, and more. Every topic has live sliders and animated feedback.

How to use it: Use this as a companion while studying. When a concept feels abstract, come here and interact with it for 5 minutes. The gradient descent 3D surface visualization alone is worth the bookmark.
// Best For
ML practitioners studying or revising the classic algorithms — this is the interactive companion to any ML textbook or course.
03 A Visual Intro to Machine Learning (R2D3) → OPEN
Probability Scrollytelling
The most beautiful scrollytelling ML explainer ever made.
A landmark piece of interactive data journalism. As you scroll, animated D3 visualizations explain how decision trees split data and why the bias–variance tradeoff exists — in a way that sticks. Part 1 covers decision trees. Part 2 dives deep into model generalization.

How to use it: Read both parts in one sitting. Don't rush — let each animation complete before scrolling. This is the piece that convinces people visual learning is not just aesthetic, it's faster and deeper.
// Best For
Anyone who has heard the terms "overfitting" and "bias–variance tradeoff" but hasn't truly internalized what they mean geometrically.
04 Seeing Theory (Brown Univ.) → OPEN
Probability Statistics Interactive
Probability and statistics, made visceral and interactive.
Brown University's visual introduction to probability and statistics. Covers basic probability, compound probability, probability distributions, frequentist inference, Bayesian inference, and regression — all through beautiful, manipulable visualizations. The Bayesian inference chapter is especially relevant for ML practitioners.

How to use it: Work through the Bayesian Inference and Regression chapters first if you're an ML engineer. These are the mathematical foundations that explain why probabilistic models behave the way they do.
// Best For
Engineers who want to understand the probabilistic foundations that underpin Bayesian classifiers, uncertainty estimation, and statistical learning theory.
05 ML Visualized (Full Course) → OPEN
Neural Nets Course
A full visual ML course — chapter by chapter, concept by concept.
A structured interactive course that walks through ML from the ground up, with the perceptron and forward propagation visualized live. Unlike single-topic tools, this is a coherent curriculum — each chapter builds on the previous, and every abstract concept has a paired interactive demo you manipulate directly.

How to use it: Start at Chapter 1 and work sequentially. Use the perceptron chapter to confirm your mental model of forward pass, then continue to backprop and beyond.
// Best For
Self-learners who want a structured curriculum with visuals built in — not just isolated demos but a coherent learning path from perceptron to deeper networks.
🖼️ Computer Vision
3 Tools to Understand CNNs Deeply

Convolutional networks are notoriously hard to build intuition for — until you watch the kernels slide and the feature maps activate. These tools make the whole thing click in minutes.

C1 CNN Explainer (Poloclub) → OPEN
CNN Live Model Open Source
Upload an image. Watch it flow through a live CNN — kernel by kernel.
A live TinyVGG CNN runs in your browser. You upload an image and watch it pass through convolution layers, activation functions, pooling, and finally softmax — with animated kernel slides showing the exact dot-product operation at each step. Click any neuron to expand it and see its activation computed live.

How to use it: First, watch the default image flow through. Then upload something unusual — a face, a meme — and notice which feature maps activate. Click the intermediate neurons to expand the convolution math. This is where "I know how CNNs work" becomes "I've seen how CNNs work."
// Best For
Anyone who has read about convolutions but still isn't 100% sure what a feature map actually contains or why pooling helps. This closes that gap in one session.
⎇ GitHub Repo
C2 Feature Visualization (Distill) → OPEN
CNN Research
What does a neuron in a neural network actually look for?
Distill's landmark interactive article on feature visualization — the technique of finding the input image that maximally activates a given neuron. You can explore what each layer of a trained network is actually detecting, from simple edges in early layers to complex textures and object parts in deeper layers.

How to use it: Read the article fully — don't skip to the visualizations. The methodology section explains why naive feature visualization fails, and the techniques section shows how researchers actually get interpretable results. Then explore the interactive neuron grids.
// Best For
ML engineers who want to understand interpretability at the neuron level — essential context for any work in model debugging, explainability, or computer vision research.
C3 Activation Atlas (OpenAI + Distill) → OPEN
CNN Research
A global map of what an entire vision network has learned.
By running feature visualization on millions of activations across an InceptionV1 network, Distill and OpenAI created a browsable "atlas" of every concept the network has encoded. You can zoom into regions representing curves, textures, animal parts, objects — and see how the network's concept space is organized.

How to use it: Click through the interactive atlas and explore the high-confidence regions. This gives you the highest-level view of what it means for a network to "understand" visual concepts — it's not a lookup table, it's a continuous learned space.
// Best For
Senior ML engineers and researchers interested in model interpretability, neural network internals, and the frontier of mechanistic understanding of deep vision models.
🤖 Transformers
4 Tools to Master the Transformer Architecture

Transformers are the architecture behind every frontier LLM. These tools take you from "I've read Attention is All You Need" to actually watching Q·K·V compute, attention heads activate, and logits emerge — live in your browser.

T1 Transformer Explainer (Poloclub) → OPEN
LLM Live Model Open Source
A live GPT-2 runs in your browser. Type text. Watch the math happen.
GPT-2 (small, 124M params) runs entirely client-side via ONNX Runtime. Type your own prompt and watch each token flow through: embedding → positional encoding → 12 transformer blocks (each with multi-head attention + MLP) → softmax → next token. Every numerical value updates live. Expand any attention head to see the Q·K·V computation. Adjust temperature and watch how the output distribution changes.

How to use it: Type a short prompt and pause at each layer to read the tooltip. Expand one attention head and trace how a single token attends to others. Then change the temperature from 0.1 to 2.0 and compare the probability distributions — this is the fastest way to internalize what temperature actually does.
// Best For
AI engineers who understand the concept of transformers but want to see the actual computational flow — tokens, weights, and probabilities — rather than just a diagram of boxes and arrows.
⎇ GitHub Repo
T2 LLM Visualization (bbycroft) → OPEN
LLM 3D Interactive
A 3D walkthrough of every weight matrix in a GPT-style LLM.
An incredibly detailed 3D interactive visualization of a full GPT-nano architecture. You can step through every operation — token embedding, positional encoding, layer norms, attention (QKV projections, softmax, output), residual streams, MLP (GELU activation), and unembedding — with the actual matrix dimensions and a running explanation. One of the most impressive educational visualizations in the ML space.

How to use it: Use the step-through mode to advance one operation at a time. Read the sidebar explanation alongside each step. This is the visualization that makes "residual stream" click — you literally watch the vector being modified at each layer.
// Best For
Engineers preparing for senior AI interviews, LLM systems design work, or anyone studying mechanistic interpretability — the residual stream view is uniquely valuable here.
T3 The Illustrated Transformer (Jay Alammar) → OPEN
LLM Visual Essay
The foundational visual essay on how the Transformer works. Read by millions.
Jay Alammar's illustrated deep dive into the Transformer from the original "Attention is All You Need" paper. Covers self-attention, multi-head attention, positional encoding, encoder–decoder architecture, and how the full model processes sequences — entirely through custom hand-drawn animated diagrams. This is the reference that Stanford, MIT, and CMU direct their students to.

How to use it: Read this before the interactive tools — it gives you the conceptual scaffold. Then revisit after using Transformer Explainer and bbycroft's viz to confirm your understanding clicked. Also read his Illustrated GPT-2 and Illustrated BERT posts afterward.
// Best For
The go-to pre-reading before any transformer implementation project, LLM course, or system design interview involving attention-based architectures.
T4 Transformer Circuits (Anthropic) → OPEN
LLM Research
Anthropic's research thread on reverse-engineering transformers as algorithms.
The living research hub from Anthropic's interpretability team. Articles explore mechanistic interpretability — the project of understanding transformers not as black boxes but as explicit algorithms with identifiable circuits. Includes interactive pieces on induction heads, attention patterns, superposition, and features as geometric structures in activation space.

How to use it: Start with the "A Mathematical Framework for Transformer Circuits" and "In-Context Learning and Induction Heads." These are technically dense — treat them like research papers, not tutorials. Extremely high-value for AI engineers interested in what's happening inside frontier models.
// Best For
Senior AI engineers, researchers, and anyone serious about model interpretability, safety, or building genuine mechanistic understanding of LLMs beyond the surface-level architectural description.
🎨 Generative AI
3 Tools for GANs & Diffusion

Generative models are among the hardest to build intuition for — the training dynamics of GANs and the denoising process of diffusion models are genuinely counter-intuitive. These tools make them observable.

G1 GAN Lab (Poloclub) → OPEN
GAN Live Model Open Source
Watch a generator and discriminator battle in real time — on data you draw.
Draw your own 2D data distribution, then watch a miniature GAN train live in the browser. You can see the generator's distribution evolve to match yours, the discriminator's decision boundary shift, and the training loss curves update in real time. Supports pausing, stepping one iteration at a time, and experimenting with learning rates and architectures.

How to use it: Start with the default ring dataset. Pause training every few hundred steps and inspect what the generator has learned. Then draw a difficult distribution (a star, disconnected clusters) and watch mode collapse happen. This single experience explains mode collapse better than any lecture.
// Best For
Anyone trying to understand GAN training dynamics, mode collapse, and why training stability is so difficult — in a hands-on way rather than through loss curve post-mortems.
⎇ GitHub Repo
G2 Diffusion Explainer (Poloclub) → OPEN
Diffusion Interactive Open Source
How Stable Diffusion turns a text prompt into an image — visualized step by step.
An interactive walkthrough of the full Stable Diffusion pipeline: text encoding via CLIP, the forward noising process, U-Net architecture for denoising, and how classifier-free guidance steers generation. You can watch the denoising happen step by step and see how the latent representation evolves from pure noise to a structured image.

How to use it: Step through each denoising iteration slowly and note how the image structure emerges. Then change the guidance scale and compare — this is the fastest way to understand what CFG actually does to the generation process.
// Best For
AI engineers working with or fine-tuning diffusion models who want to understand what's happening in the denoising loop beyond the high-level description of "we reverse noise."
G3 The Illustrated Stable Diffusion → OPEN
Diffusion Visual Essay
Jay Alammar's visual companion to understanding latent diffusion models.
Alammar's signature illustrated-essay treatment applied to Stable Diffusion. Covers the VAE encoder/decoder, the CLIP text encoder, the U-Net denoising architecture, and the entire generation pipeline — all through step-by-step diagrams that show tensor shapes and data flow explicitly.

How to use it: Read this before using the Diffusion Explainer — it gives you the vocabulary (latent space, timestep conditioning, cross-attention for text) that makes the interactive tool significantly more useful.
// Best For
Engineers who want to understand latent diffusion models well enough to fine-tune, prompt-engineer, or extend them — not just use them as black-box APIs.
📐 Distill.pub
5 Must-Read Interactive Research Articles

Distill.pub is the gold standard for interactive ML research writing. Peer-reviewed, interactive, and built by the same researchers who created the models being explained. Every article below is worth your full attention.

D1 How to Use t-SNE Effectively → OPEN
Dimensionality Reduction Interactive
Kills a dozen common t-SNE misconceptions with live sliders.
Drag the perplexity and iteration sliders and watch how t-SNE plots change — including cases where the clusters it shows are completely meaningless. This article is essential for anyone who has ever interpreted a t-SNE plot in a paper or presentation and assumed the clusters meant something structural.
// Best For
Data scientists and ML engineers who regularly use t-SNE or UMAP for embedding visualization and want to avoid the common misinterpretations that pollute ML papers.
D2 Why Momentum Really Works → OPEN
Optimization Interactive
The intuition for SGD, momentum, and Adam — on interactive loss landscapes.
Watch gradient descent with and without momentum navigate 3D loss surfaces interactively. This article builds deep intuition for why adaptive optimizers exist, what momentum actually does to gradient updates geometrically, and why saddle points are more problematic than local minima in high-dimensional spaces.
// Best For
Engineers who choose between SGD, Adam, AdamW, and RMSprop by instinct rather than understanding — this article replaces instinct with genuine mechanistic understanding.
D3 Attention and Augmented RNNs → OPEN
Attention Visual Essay
The foundational visual overview of attention — written before transformers took over.
Chris Olah and Shan Carter's visual overview of the attention mechanism, Neural Turing Machines, memory networks, and differentiable programming — written in 2016. Reading this gives you the historical context that makes modern transformer attention feel like a natural evolution rather than a mysterious invention.
// Best For
Understanding where the attention mechanism came from and why it was designed the way it was — giving you deeper context for every transformer-based architecture decision.
D4 Visualizing Neural Network Loss Landscapes → OPEN
Neural Nets Optimization
What does the loss surface of a deep network actually look like?
An interactive exploration of linear dimensionality reduction applied to neural network loss landscapes — showing how shallow vs deep networks and different architectural choices create qualitatively different optimization terrains. Reframes "why does this network train better" into a geometric question you can see.
// Best For
ML engineers debugging training instability, choosing architectures, or trying to build intuition for why batch size, learning rate, and depth interact the way they do.
D5 Distill.pub — Full Archive → OPEN
Research Journal All Topics
The complete journal — 40+ peer-reviewed interactive ML articles.
Distill is on hiatus since 2021 but the archive is timeless. Every article is interactive, peer-reviewed, and written by researchers at Anthropic, Google Brain, OpenAI, and leading universities. The circuits thread, feature visualization, and neural network interpretability articles are landmark pieces for anyone serious about understanding deep learning beyond the API level.
// Best For
Bookmark this and return to it whenever you hit a concept that still feels like magic. There's a Distill article for almost every foundational concept in modern deep learning.

R4 helblazer811 / ManimML → OPEN
Python Library Animation
Create 3Blue1Brown-style ML animations with Python — CNNs, VAEs, attention, and more.
A Python library built on top of Manim (the animation engine Grant Sanderson uses for 3Blue1Brown) specifically for ML concept animation. Pre-built scene components for neural networks, CNNs, attention mechanisms, VAEs, and custom architectures. Generate publication-quality or YouTube-quality animations from Python code.

How to use it: Install via pip. Start with the example scenes in the repo. This is the tool for creating animated ML diagrams for talks, courses, or YouTube — not for interactive exploration.
// Best For
AI content creators, researchers preparing conference talks, and educators who want to produce animated explanations of ML architectures without learning a full animation tool from scratch.

Full source for GAN Lab — extend it for your own distribution or architecture.
Open-source TensorFlow.js + D3.js codebase behind GAN Lab. The architecture is clean enough to extend — you can swap in different generator/discriminator designs or replace the 2D distribution with your own to build custom GAN training visualizations.

Researchers or educators who want to build custom GAN training visualizations for specific distribution types or architecture variants beyond the defaults in the hosted tool.
