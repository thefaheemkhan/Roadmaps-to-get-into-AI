Path A — Builder's Perspective
Applied / Research Scientist
01
Foundations & Math
4–6 weeks ✓ ▼
Linear Algebra & Calculus
  • Matrix operations, eigendecomposition, SVD
  • Gradients, Jacobians, chain rule
  • Backpropagation derivation from scratch
  • Computational graphs, automatic differentiation
Probability & Statistics
  • Probability distributions (Normal, Categorical, Dirichlet)
  • MLE, MAP estimation, Bayesian inference
  • KL divergence, entropy, cross-entropy loss
  • Monte Carlo methods, importance sampling
Deep Learning Fundamentals
  • Optimizers: SGD, Adam, AdamW, Lion
  • Regularization: dropout, weight decay, layer norm
  • Learning rate schedules: warmup, cosine, linear decay
  • Batch norm vs layer norm vs RMS norm
📄 Key Papers
📄Adam Optimizer (Kingma 2014) 📄Layer Normalization (Ba 2016)
🔨 Projects
🔨Build autograd engine (like micrograd) 🔨Implement Adam from scratch + test convergence
🔗 Resources
🔗3Blue1Brown Neural Nets 🔗fast.ai Part 2 🔗micrograd by Karpathy
02
Transformer Architecture
3–4 weeks ✓ ▼
Attention Mechanisms
  • Self-attention: queries, keys, values
  • Multi-head attention, scaled dot-product
  • Causal masking, bidirectional attention
  • KV-cache, grouped-query attention (GQA), MQA
  • Flash Attention I & II — memory-efficient implementation
Architecture Variants
  • Encoder-only: BERT, RoBERTa, DeBERTa
  • Decoder-only: GPT series, LLaMA, Mistral, Gemma
  • Encoder-decoder: T5, BART, mT5
  • Mixture of Experts (MoE): Mixtral, Switch Transformer
  • State Space Models: Mamba, RWKV
Positional Encoding
  • Sinusoidal (original), learned absolute positions
  • Relative: RoPE, ALiBi, T5 relative bias
  • Context length extrapolation techniques
📄 Key Papers
📄Attention Is All You Need (2017) 📄Flash Attention (Dao 2022) 📄RoPE (Su 2021) 📄Switch Transformer MoE (2021)
🔨 Projects
🔨Implement GPT-2 from scratch in PyTorch 🔨Train character-level transformer on Shakespeare 🔨Visualize attention heads on your trained model
🔗 Resources
🔗nanoGPT by Karpathy 🔗Annotated Transformer 🔗LLM Visualization (3D)
03
Tokenization & Data
2–3 weeks ✓ ▼
Tokenization Strategies
  • BPE (Byte-Pair Encoding) — GPT family
  • WordPiece — BERT family
  • SentencePiece, Unigram language model
  • Byte-level tokenization (GPT-2 style)
  • Vocabulary design trade-offs & fertility metrics
Pretraining Data Pipeline
  • Web scraping: CommonCrawl, C4, FineWeb
  • Deduplication: MinHash LSH, exact n-gram matching
  • Quality filtering: fastText classifier, perplexity filtering
  • Data mixing ratios & domain balance
  • Datasets: The Pile, RedPajama, SlimPajama, DOLMA
📄 Key Papers
📄The Pile (Gao 2020) 📄SlimPajama Dataset (2023) 📄FineWeb (HuggingFace 2024)
🔨 Projects
🔨Train BPE tokenizer on custom domain corpus 🔨Build a data quality filtering pipeline 🔨Compare tokenizers: analyze token fertility on Hindi/code
🔗 Resources
🔗HuggingFace Tokenizers 🔗Tiktokenizer (visual)
04
Pretraining
4–6 weeks ✓ ▼
Training Objectives
  • Causal LM (CLM) — next token prediction
  • Masked LM (MLM) — BERT-style
  • Span corruption — T5-style
  • Contrastive objectives (SimCSE, CLIP)
Distributed Training
  • Data parallelism, tensor parallelism, pipeline parallelism
  • ZeRO stages 1/2/3 (DeepSpeed)
  • FSDP — PyTorch Fully Sharded Data Parallel
  • Gradient checkpointing, mixed precision (fp16/bf16)
  • Gradient accumulation & clipping strategies
Scaling Laws
  • Kaplan scaling laws (OpenAI)
  • Chinchilla compute-optimal training
  • Emergent abilities — phase transitions at scale
  • Inference-time scaling (o1-style)
📄 Key Papers
📄Scaling Laws (Kaplan 2020) 📄Chinchilla (Hoffmann 2022) 📄ZeRO (Rajbhandari 2020) 📄GPT-3 (Brown 2020)
🔨 Projects
🔨Pretrain 125M model on OpenWebText 🔨Reproduce scaling law plot: loss vs compute 🔨Train with FSDP on multi-GPU setup
🔗 Resources
🔗nanoGPT 🔗DeepSpeed 🔗GPT-NeoX
05
Fine-tuning & Alignment
4–5 weeks ✓ ▼
Supervised Fine-Tuning (SFT)
  • Instruction tuning: FLAN, Alpaca, WizardLM
  • Chat formatting: ChatML, Llama chat, Alpaca format
  • Continual pretraining for domain adaptation
  • Multi-task fine-tuning strategies
Parameter-Efficient Fine-Tuning (PEFT)
  • LoRA — low-rank adaptation of weight matrices
  • QLoRA — quantized LoRA for consumer GPUs
  • DoRA — weight decomposition enhanced LoRA
  • Prefix tuning, prompt tuning, IA³
  • Adapter layers (Houlsby et al.)
Alignment & RLHF
  • Reward model training from human preferences
  • PPO for RLHF — Proximal Policy Optimization
  • GRPO — Group Relative Policy Optimization (DeepSeek)
  • DPO — Direct Preference Optimization (offline)
  • IPO, KTO — alternative preference methods
  • Constitutional AI (CAI), RLAIF (AI feedback)
📄 Key Papers
📄LoRA (Hu 2021) 📄InstructGPT / RLHF (Ouyang 2022) 📄DPO (Rafailov 2023) 📄QLoRA (Dettmers 2023) 📄Constitutional AI (Anthropic 2022)
🔨 Projects
🔨Fine-tune LLaMA-3 on custom dataset via QLoRA 🔨Train reward model + run DPO on your SFT model 🔨Create preference dataset using GPT-4 as labeler
🔗 Resources
🔗TRL (HuggingFace) 🔗Unsloth 🔗Axolotl
06
Optimization & Inference
3–4 weeks ✓ ▼
Quantization
  • Post-training quantization: GPTQ, AWQ, GGUF
  • Quantization-aware training (QAT)
  • 1-bit / 2-bit models (BitNet b1.58)
  • Activation quantization, KV-cache quantization
Pruning & Distillation
  • Structured pruning (head pruning, layer dropping)
  • Unstructured pruning (SparseGPT)
  • Knowledge distillation (teacher-student)
  • Speculative decoding — draft + verify
Efficient Serving
  • PagedAttention — vLLM's memory management
  • Continuous batching, dynamic batching
  • Tensor parallelism for multi-GPU serving
  • Decoding strategies: greedy, beam, nucleus, top-k
  • Structured generation with constrained decoding
📄 Key Papers
📄GPTQ (Frantar 2022) 📄AWQ (Lin 2023) 📄vLLM/PagedAttention (Kwon 2023) 📄BitNet b1.58 (2024)
🔨 Projects

Prompt Engineering
2–3 weeks ✓ ▼
Core Techniques
  • Zero-shot, one-shot, few-shot prompting
  • Chain-of-thought (CoT) — "think step by step"

  • Self-consistency: majority vote over multiple samples
  • XML/JSON structured output prompting
  • Role prompting, persona, system prompt design
Prompt Optimization
  • DSPy — automatic prompt optimizer
  • Meta-prompting & self-critique loops
  • A/B testing prompts with eval datasets
  • Prompt versioning & management

04
RAG — Retrieval-Augmented Generation
3–4 weeks ✓ ▼
Naive RAG Pipeline
  • Chunking: fixed-size, recursive, semantic, markdown-aware
  • Indexing — embedding + storing chunks
  • Retrieval — ANN search + top-k selection
  • Augmented generation with context
  • Context window management for long docs
Advanced RAG
  • HyDE — hypothetical document embeddings
  • Self-RAG — self-reflective retrieval
  • CRAG — corrective RAG with web search fallback
  • Multi-query retrieval (query expansion)
  • Parent-child & sentence-window chunking
  • GraphRAG — Microsoft's graph-based approach
RAG Evaluation
  • Faithfulness — is the answer grounded?
  • Answer relevancy — does it answer the question?
  • Context recall — was right info retrieved?
  • RAGAS framework for automated evaluation
📄 Key Papers
📄RAG (Lewis 2020) 📄Self-RAG (Asai 2023) 📄GraphRAG (Edge 2024) 📄RAG Survey (Gao 2023)
🔨 Projects
🔨PDF Q&A chatbot with RAGAS evaluation 🔨Multi-hop RAG over Wikipedia for complex questions 🔨Codebase Q&A with GraphRAG (repo-level understanding)
🔗 Resources
🔗LlamaIndex 🔗LangChain 🔗RAGAS
05
Fine-Tuning for Engineers
3–4 weeks ✓ ▼
When to Fine-Tune vs RAG
  • Style/format/persona → fine-tune
  • Factual knowledge → RAG (cheaper, updatable)
  • Dataset quality over quantity — curation strategies
  • Synthetic data generation: Evol-Instruct, Self-Instruct
Practical Fine-Tuning
  • QLoRA on consumer GPUs with Unsloth
  • Axolotl config-based fine-tuning
  • Merging adapters: linear merge, SLERP, DARE
  • ORPO, SimPO — combined SFT + preference
Managed Platforms
  • OpenAI fine-tuning API (GPT-3.5, GPT-4o mini)
  • HuggingFace AutoTrain
  • Modal.com, RunPod for GPU compute
  • Together AI, Anyscale fine-tuning
🔨 Projects
🔨Fine-tune Mistral-7B on customer support (QLoRA) 🔨Generate synthetic dataset with GPT-4 + evaluate quality 🔨Merge two fine-tuned LoRA adapters with SLERP
🔗 Resources
🔗Unsloth 🔗Axolotl 🔗OpenAI Fine-tuning

Tool Use & Function Calling
